FAQs

Frequently Asked Questions

Read through FAQs for further information.

  • Is this a tool or a concept?

    Smart Foundry is positioned as a platform. Capgemini has leveraged its intelligent integration framework to seamlessly fit in the best of the value stream management capabilities from Plutora and the multi-premise deployment capabilities of Nutanix’s automated application management framework, to create an intelligent, multi-functional, domain-centric platform – Smart Foundry. In Summary, Capgemini’s Smart Foundry provides a powerful, domain-centric, multi-premise, cloud agnostic and a secure environment for efficient, scalable testing. It is your one stop shop for all your testing, test environment and test data needs.

  • Why choose Smart Foundry over any single cloud platform?

    Smart Foundry used Nutanix’s Calm features to enable multi-premise deployment of workloads. This means that an organisation is not tied to a single cloud platform or on-premise compute. They can make cost-based decisions to deploy the code to the right cloud platform. This is one of the key features of Smart Foundry. Multi-Premise includes Public, Private or On-Premise facilities. If a customer has only one cloud platform, Smart Foundry significantly reduces deployment effort using blueprints and container solutions.

  • How can Smart Foundry be installed in client environment within Capgemini facility?

    The engine of the solution comprises of Plutora and Nutanix Calm. Plutora is a SaaS solution so this can be accessed by the customer if the network connectivity along with necessary security policies are implemented. Nutanix Calm can be accessed through an AWS instance or through an on-premise instance. The network connectivity and adequate security policies are vital for safe and secure communication. The accompanying features of Smart Foundry can be a combination of on-premise, SaaS or a cloud instance depending on whether a tool is already available at the customer premise.

  • Does it provide reporting solutions?

    Smart Foundry has elaborate reporting solutions. This can be accessed via the value stream management module of the platform. However, although integrated, it is also possible to gain specific information from the individua tools already integrated within the Smart Foundry ecosystem.

  • What level of programming/ technical skillset does one need to have to benefit out of Smart Foundry?

    Smart Foundry is generally user friendly and UI based. Hence the level of programming skills is limited to operate most parts of the platform. However, to create environment blueprints for the first time, we recommend that you have strong Python or similar scripting skills. General awareness of infrastructure and server technologies are required to comprehend the overall solution. Also, it may be important to understand the concept of container and service virtualisation solutions along with test data management so this solution can be effectively implemented. Working understanding of the tools involved will be necessary to operate the tools in the eco-system depending on the type of tool used.

  • How can one subscribe to different platforms, solutions to build Smart Foundry ecosystem?

    Smart Foundry can be implemented in a modular construct or as an entire eco-system. This depends on the requirements and the challenges that the customer is trying to address. Capgemini would recommend deploying this platform to address specific solutions in a planned approach as there will be an initial custom integration required based on client – specific use cases.

  • Can the environment be reused within multiple work streams in the same project?

    Yes. The use of container and blueprint concepts will enable full flexibility of creating and deploying multiple application environments. As Smart Foundry provides the capability to also integrate service virtualisation, this makes it a compelling proposition for running parallel releases or work streams that has dependency on the same set of application environments for test and development purposes.
